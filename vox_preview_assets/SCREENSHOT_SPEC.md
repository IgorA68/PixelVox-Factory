# Screenshot Spec

This file defines the minimal capture standard for repository-facing screenshots in `vox_preview_assets/`.

The goal is consistency, not visual overproduction. Screenshots should help a GitHub visitor understand what the project can generate within a few seconds.

## Required Output Style

- Use PNG for still images.
- Keep backgrounds neutral and low-contrast.
- Keep lighting readable and consistent across the set.
- Prefer a fixed family of camera angles for comparable assets.
- Avoid text overlays inside the image unless the image is specifically for GUI walkthroughs.

## Canonical Screenshots

The current repository-facing set should prioritize these files:

- `house.png`
- `realistic_tree_vox.png`
- `vehicle.png`
- `tower.png`
- `cave.png`
- optional: `gui-launcher.png`
- optional: `showcase-grid.png`

## Framing Guidance

- Fill most of the frame with the model while keeping a small border around silhouettes.
- Keep the object centered unless an off-center composition improves readability.
- Avoid extreme perspective distortion.
- For terrain-like assets, prefer a camera angle that makes elevation changes obvious.

## Consistency Rules

- Use the same capture approach for all README showcase stills in one release.
- Do not mix diagnostic preview renders with final curated screenshots.
- If a screenshot replaces an existing canonical file, keep the same filename unless the README is updated in the same change.

## GUI Screenshot Guidance

If `gui-launcher.png` is included:

- show one valid blueprint selected;
- keep the important controls visible;
- avoid personal desktop clutter or unrelated windows;
- capture at a readable size without blurry scaling.

## Release Check

Before tagging a release, confirm that:

- every screenshot referenced by the README exists;
- the screenshot set matches the current demo set closely enough to avoid misleading users;
- no temporary or autogenerated-only files are being presented as final showcase media.
