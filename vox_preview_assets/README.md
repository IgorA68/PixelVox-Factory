# Preview Assets

This folder is reserved for screenshots, GIF previews, and other visual assets used by the repository documentation.

It now contains two asset categories:

- autogenerated preview PNG files produced by the Python pipeline, which include a mode suffix such as `house-isometric.png` or `hills-top.png`;
- final curated screenshots for README/publication, which keep canonical names such as `house.png`, `vehicle.png`, or `showcase-grid.png`.

## Intended Files

- `showcase-grid.png`: recommended top-of-README overview image.
- `gui-launcher.png`: GUI screenshot used in the README quick-start section.
- `house.png`: final MagicaVoxel README screenshot for the baseline house example.
- `realistic_tree_vox.png`: final MagicaVoxel README screenshot for the organic showcase.
- `vehicle.png`: final MagicaVoxel README screenshot for the rover showcase.
- `hills-isometric.png`: terrain showcase image.
- `house-isometric.png`: simple baseline example for the quick-start section.
- `rocket-isometric.png`: stylized prop showcase image.
- `realistic_tree_vox-isometric.png`: organic showcase image.
- `vehicle-isometric.png`: rover showcase image.
- `tower-isometric.png`: architectural showcase image.
- `cave-isometric.png`: negative-space showcase image.
- `build-process.gif`: optional animated preview of the build workflow.

## Recommended First Shot List

- `showcase-grid.png`: a curated multi-model overview using `vehicle`, `realistic_tree_vox`, `tower`, and `cave`.
- `gui-launcher.png`: the launcher with one blueprint selected and metadata visible.
- `house.png`: the simplest readable starter example for quick-start documentation.
- `realistic_tree_vox.png`: the strongest organic example.
- `vehicle.png`: the strongest co-created hard-surface example.

If you want one more still image, `cave.png` is a better sixth shot than `rocket.png` because it shows negative space and interior reading more clearly.

## Capture Guidance

- Use consistent framing and camera angles across showcase renders.
- Prefer the same palette mode and lighting style for all screenshots.
- Keep MagicaVoxel on the `pal0` baseline or a clearly documented derivative when capturing showcase images.
- Keep image backgrounds neutral so the voxel silhouettes read clearly.
- Export lossless PNGs for still images when possible.
- Only add GIFs when they communicate something static screenshots cannot.

## Publication Note

These assets are intentionally separate from generated `.vox` files in `vox_models/`.
Keep preview media curated and repository-facing rather than treating this folder as a general export dump.

Use the unsuffixed names for final publication screenshots and the mode-suffixed names for autogenerated preview outputs.

## Batch Generation

To regenerate the current showcase PNG set directly into this folder, run:

```bash
python -m vox_art.build_preview_assets --preview-mode isometric --preview-scale 8
```

This batch command regenerates only the PNG showcase files in this folder and does not rewrite generated `.vox` files in `vox_models/`. Generated preview files include the selected preview mode in their filename, such as `house-isometric.png` or `hills-top.png`.

For a more diagnostic contact-sheet style, switch to orthographic mode instead.
